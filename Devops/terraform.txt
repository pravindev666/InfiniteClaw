to run a code for infrastructure
go to terraform registry
to create a vpc
*to login to console--
i have to select a provider ---Scroll down select AWS
Version---AWS Documenation Version--Choose 1 month before version
*AWS Provider
Copy the first step

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

Second Step:Select Region

provider "aws" {
  region = "ap-southeast-1"
}

Third Step:
Scroll the left side and select VPC at last
Dont touch the default vpc

resource:aws_vpc
reference name:Nickname(Can Be Anything)
Basic usage with tags:

resource "aws_vpc" "nickname" {
  cidr_block       = "10.0.0.0/16"
  instance_tenancy = "default"

  tags = {
    Name = "NICKNAME"
  }
}

Fourth Step:
Resource: aws_subnet
Subnets:
Basic Usage
PUBLIC SUBNET:

resource "aws_subnet" "publicnickname" {
  vpc_id     = aws_vpc.nickname.id
  cidr_block = "10.0.1.0/24"

  tags = {
    Name = "PUBLICNICKNAME"
  }
}

PRIVATE SUBNET:

resource "aws_subnet" "privatenickname" {
  vpc_id     = aws_vpc.nickname.id
  cidr_block = "10.0.2.0/24"

  tags = {
    Name = "PRIVATENICKNAME"
  }
}

Fifth Step:
Internet Gateway:
Resource: aws_internet_gateway

resource "aws_internet_gateway" "nicknameigw" {
  vpc_id = aws_vpc.nickname.id

  tags = {
    Name = "NICKNAMEIGW"
  }
}

Sixth Step:
Route Table:
Resource: aws_route_table

PUBLIC ROUTE TABLE:
resource "aws_route_table" "nicknamepubrt" {
  vpc_id = aws_vpc.nickname.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.nicknameigw.id
  }

  tags = {
    Name = "PUBLICRT"
  }
}

PRIVATE ROUTE TABLE:
NOTE:TO CONNECT NAT GATEWAY

resource "aws_route_table" "nicknameprirt" {
  vpc_id = aws_vpc.nickname.id

  tags = {
    Name = "PRIVATERT"
  }
}

Seventh Step:
NAT GATEWAY:
Resource: aws_nat_gateway
